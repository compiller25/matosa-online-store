1. PROJECT OVERVIEW
────────────────────────────────────────────────────────────────────────────────

  Name:          Matosa Online Shopping (matosa-online-shop)
  Type:          React Native / Expo mobile app
  Market:        Vegetable home delivery in Dar es Salaam, Tanzania
  Audience:      Neighbors in the Matosa area (Kilimani, Dar es Salaam)
  Currency:      TZS (Tanzanian shillings)
  Languages:     Kiswahili + English
  Current state: Functional MVP — version 1.2.0

  Tech stack:
  ┌──────────────┬──────────────────────────────────────────────────────────┐
  │ Framework    │ Expo ~54.0.34, React Native 0.81.5, React 19.1.0        │
  │ Language     │ JavaScript (with unused .ts/.tsx files)                  │
  │ Navigation   │ React Navigation (Drawer + Bottom Tabs + Stack)          │
  │ State        │ Zustand 5 with AsyncStorage persistence                  │
  │ Maps         │ OpenStreetMap + Leaflet via WebView (expo-location)      │
  │ Payments     │ M-Pesa, TigoPesa, Airtel Money (reference only)         │
  │ Checkout     │ Order via WhatsApp (wa.me)                              │
  │ Notifications│ expo-notifications (local only, no Firebase)             │
  │ Backend      │ NONE — fully offline/local                              │
  └──────────────┴──────────────────────────────────────────────────────────┘

4. ADDITIONAL RECOMMENDATIONS
────────────────────────────────────────────────────────────────────────────────

  4.1 SECURITY
  ────────────────────────────────────────────────────────────────────────────
  • The admin PIN is hardcoded ("1234") — should be configurable or use
    biometric authentication.
  • User profile data is stored in plain text in AsyncStorage — consider
    encryption if any sensitive data is added.

  4.2 USER EXPERIENCE
  ────────────────────────────────────────────────────────────────────────────
  • Cart and orders are lost if the user clears app data — migrating to
    Firebase in Phase 3 solves this.
  • Green vegetable prices are hardcoded (600 TZS) — allow configuration
    from the admin panel.
  • The business WhatsApp number (255757808854) is hardcoded in
    src/utils/whatsapp.js — move to a more accessible configuration.

  4.3 PERFORMANCE
  ────────────────────────────────────────────────────────────────────────────
  • bg001.jpg (2.8MB) and background.jpg (1.7MB) — compress to <200KB.
  • Consider expo-image for caching and progressive image loading.

────────────────────────────────────────────────────────────────────────────────

5. CONCLUSION
────────────────────────────────────────────────────────────────────────────────

  The current product is a solid MVP with:

  • 17 functional screens
  • 5 stores with persistence
  • Full i18n (SW/EN)
  • Working WhatsApp integration
  • Functional admin panel
  • Delivery zone system
  • Mobile payment options (M-Pesa, TigoPesa, Airtel Money)

  Phase 1 fixes take ~1 week and remove current blockers. Phase 2 (real
  payments + push notifications + categories) turns the app into a
  product ready for the Tanzanian market.

  Recommendation: start with Phase 1 + mobile money (M-Pesa/TigoPesa/
  Airtel Money) as the first sprint — fixes critical bugs and adds the
  highest-impact feature for the local market.
